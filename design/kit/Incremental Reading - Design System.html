Incremental Reading — Design System

Open prototype

Design rationale

Before the screens — here's the thinking behind the system, so we're aligned on direction. I'm treating this as a serious, long-term knowledge environment: dense, calm, keyboard-first, with restraint as the dominant feeling.

Type — a single superfamily

IBM Plex does all three jobs and keeps everything coherent:

  • Plex Sans — all UI chrome, labels, controls
  • Plex Serif — long-form reading & card faces, so reading feels distinct from operating the tool
  • Plex Mono — metadata, intervals, counts, shortcut keys

Color — neutral with intent

A cool-neutral gray canvas carries 95% of the UI. Color appears only to mean something: priority (A–D), status (due / overdue / leech), and element type. No decorative gradients, no illustration.

Density & speed

Compact 4px spacing grid, 13.5px base UI text, tight rows. Designed so a power user processing 80 items a day never feels the chrome. Subtle borders over heavy shadows.

Keyboard-first, quietly

Every primary action has a shortcut, shown subtly on hover and in corners — present for power users, never shouting at newcomers. A command bar anchors the top of every screen.
